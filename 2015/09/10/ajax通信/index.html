C/S架构和B/S架构

C/S架构 即Client和Server，在C/S结构下，不同的服务需要安装不同的客户端软件，比如QQ，迅雷，Foxmail这种软件会越来越多，从而占用较多的系统资源，除此之外，还有弊端是：要想在别的电脑上使用Foxmail这样的服务，必须要先下载Foxmail客户端。而且软件升级后也必须要 重新下载。

B/S架构 即Brower和Server，在B/S架构下，所有的服务将通过浏览器来完成，因为基本所有的电脑都安装了浏览器。获取服务只需要输入网址，刷新网页即可。但是，B/S也有一些不利的方面，比如：操作的稳定性，流畅度方面相对较弱。

服务器类型

能提供某种服务能力的计算机称为服务器。

  • 按照服务类型分类：文件服务器 数据服务器
  • 按操作系统分类： Linux服务器 Windows服务器
  • 按应用软件分类：Apache服务器 Ngnix服务器 IIS服务器 Tomcat服务器 Node服务器

服务器的软件

使计算机具备提供某种服务能力的软件，称为服务器软件。常见的服务器软件有：

  1. 文件服务器：Server-U FileZila VsFTP
  2. 数据库服务器： Oracle MySQL PostgreSQL MSSQL
  3. 邮件服务器：Postfix Sendmail
  4. Web服务器：Apache、Nginx、IIS、Tomcat、NodeJS

HTTP服务器
即网站服务器，主要提供文档（文本，图片，视频，音频）的浏览服务，一般安装Apache,Nginx服务器软件。HTTP服务器结合某一编程语言处理业务逻辑，由此进行开发，称为服务端开发。
常见的在服务端运行的编程语言有：PHP Jsp Asp Python Ruby

网络基础

  • IP地址
    所谓IP地址就是给每个连接在互联网上的主机分配的一个32位地址。(就像每部手机能正常通话需要一个号码一样)
    查看本机IP地址 ping、ipconfig、ifconfig
  • 域名
    IP地址基于数字，不便于记忆，于是使用域名来代替IP地址。
  • DNS服务
    DNS公网服务器，记录了IP地址和域名的映射关系
    地址查找优先级:
    本机的host文件 —-> DNS服务器
  • 端口
    用来确定应用服务的类型，不同的端口对应不同的服务
    查看端口的占用情况： netstat -an

网络传输协议

指的是客户端和服务端间进行通信时的约束和规范，客户端和服务端的数据交互不是杂乱无章的，需要遵循一定的规范来进行。

常见的协议

1、HTTP、HTTPS 超文本传输协议
2、FTP 文件传输协议
3、SMTP 简单邮件传输协议

HTTP协议

即超文本传输协议，网站是基于http协议的，例如网站的css``html图片等都是基于http协议进行通信（传输）的。
HTTP协议具体是指客户机到服务器的请求和服务器到客户机的响应进行了一系列的约束和规范。
所以所谓的http协议主要是由请求和响应构成的。

常见的请求方法

POST GET PUT DELETE

请求报文

请求的报文是由客户端发起的，它的规范是：请求行 请求头 请求主体

  1. 请求行
    由请求方式、请求URL、和协议版本组成

  2. 请求头
    Host 请求的主机名
    Cache-Control: max-age=0 控制缓存
    Accept: */* 接受文档的MIME类型
    User-Agent: 很重要 浏览器和系统的信息
    Referer: 从哪个URL跳转过来的
    Accept-Encoding: 可接受的压缩格式
    注意：
    当请求的方式是POST的时候，请求头里会设置

当请求的方式是get的时候，不需要设置

  1. 请求主体
    也就是传递给服务器的数据

响应报文

响应报文是由服务端发出的，它的规范是：响应行（状态行） 响应头 响应主体

  1. 状态行
    由协议版本号、状态码和状态信息构成
    例如：HTTP/1.1 200 OK
    关于状态码
    常见的状态码有：
    200 代表响应成功 304 代表文档未修改 403 没有权限 404 未找到 500 服务器错误

  2. 响应头

Date: 响应的时间（服务端的时间）
Server: 服务器的信息
Content-Length: 响应主体的长度
Content-Tyep: 响应资源的MIME类型
注意：
MIME是标识文件类型的，文件的后缀并不能正确的标识文件的类型。客户端与服务器之间的数据传输都是以字节的形式进行的，咱们可以理解是以”文本的形式”传输的，这时浏览器就需要知道该怎么解析这些文本形式的数据，MIME就是明确告诉浏览器该如何处理这些数据。

  1. 响应主体
    即服务端返回给客户端的内容

Ajax编程

Ajax即Asynchronous Javascript And XML 不是新的语言，而是对现有技术的综合利用。本质上是在HTTP协议的基础上以异步的方式与服务器进行通信。

  • 异步
    指的是某段程序在执行时不会阻塞其他程序的执行，其表现形式为程序的执行顺序不依赖程序本身的书写顺序，相反则为同步。
    异步的优势在于：不阻塞程序的执行，从而提升整体的执行效率。
  • XMLHttpRequest
    浏览器的内建对象，用于在后台和服务器通信，由此我们便可以实现对网页的部分更新，而不是刷新整个页面。

分析：
XMLHttpRequest的方法和Http请求的3个组成部分正好相对应

  1. 请求行
    xhr.open('get','index.php');
  2. 请求头
    xhr.setRequestHeader('Content-Type','text/html')
  3. 请求主体
    xhr.send(null);
