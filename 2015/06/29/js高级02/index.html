构造函数、实例对象和原型的关系

  • 构造函数 -> 抽象的模板
  • 实例对象 -> 具体的对象

  • 原型 -> 共享数据和继承

    构造函数、实例对象与原型之间的关系：
     1.构造函数中都有原型属性prototype，该属性本质也是一个Object实例
     2.原型对象中有一个属性constructor,指向原型对象所处的构造函数
     3.实例对象中都有一个属性__proto__，该属性不是标准属性，该属性指向创建构造实例对象的构造函数中的原型对象

原型链

  • 原型链的结构可以通过重置原型对象的方式修改，从而能延长原型链
  • 继承：继承最基础的方式是通过原型链实现，本质上是代码的复用

初始继承

继承：实现代码的复用，别人的代码可以直接拿过来自己用，自己的代码还是自己的

  • 继承的第一种方式：通过修改原型的指向来实现

原型对象的补充

  • 如果原型中的方法很多，那么把方法合入到一个对象中，这时要手动修改原型中的constructor属性，让它指向所属的构造函数

原型继承的问题

  • 原型继承缺点：
    1. 没有办法在不影响所有实例的情形下，给继承过来的属性赋值(在new的方式下)
    2. 继承过来的引用类型的数据，所有的实例对象共享

call和apply的基本使用

相同点：call和apply 都可以调用函数，作用：改变所调用函数内部this的指向；区别：call传递的参数是以逗号分隔，apply传递的参数是以数组的方式

  • 在非严格模式下，普通函数中this是window，在严格模式下，普通函数中this是undefined

分析问题

数值的比较

官方对于‘ == ’的说明：如果其中一个值是true，则将其转换为1再进行比较。如果其中一个值是false，则将其转换为0再进行比较。如果一个值是对象，另一个值是数字或字符串，则将对象转换为原始值再进行比较。

总结部分

  1. 对象之间的比较是内存地址的比较，单个空对象转成布尔值是true
  2. 空数组与布尔值的比较会转换成数值的比较（空数组会转化成0）
  3. 转成false的情况：false null '' undefined 0 NaN;其他情况都转成true

in的使用

  • 声明变量的时候是否添加var差别：不使用var不会预解析，但是也会向window中添加属性

寄生式继承
