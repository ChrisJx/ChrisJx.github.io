弹窗

目前我遇到的弹窗模式有以下几种：

  1. 页面处于顶端（弹窗大小固定居中）
  2. 页面处于中间位置 （弹窗大小固定居中）
  3. 弹窗内可以进行滑动
  4. 弹窗从底部进行弹出
  5. 提示弹层

ok， 这些情况曾在jquery上面困扰了不少时间，现在进行一一解决，show code!

本来是想在 store 中的 action 中来写的，但是磊哥提醒 action 中的最好用来写一些异步的公用操作，比如：加入到购物车中

但是又学到了另外一种方式，mixins

mixins 中关于父子组件之前的通信全部使用 props 来传递，这使得在弹窗间的通信以及子弹窗之间的通信就变得相对复杂。不过既然已经使用了这种使用于 父子弹窗通信 的方式，就继续实现下去吧！

首页模块

我把首页的各个部分拆分的很细，实际上只要把主体投资平台部分拆出来就可以了。其中话题模块也可以单独拉出来，涉及到对不同终端和app版本进行处理。

  • 默认的数据类型保持不变，这个好处是不容易混乱数据是否有值的判断。

空对象的判断 JSON.stringify(data) == "{}"

空数组的判断 data.length == 0

  • 对于像首投，复投，或者话题这些要处理的数据，使用计算属性进行二次处理。

好处： 可以直接建立 抽取出来的数据 和 主数据 的依赖，不需要事先在 data 中进行声明。

  • 对于数组类 富文本的处理

其实这个部分更好的方式是放在后台进行处理，受制于editor编辑器所以可以使用两种方式。

  1. 和运营约定段落的分隔符号，比如 ‘;’ (注意英文的大小写)，并且去掉多余的空格。然后直接让后台存储 纯文本格式的内容
  2. 无所不用其极的 正则表达式
