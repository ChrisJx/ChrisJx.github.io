关于面试的整理

网络http相关

  1. ★★★HTTP状态码有哪些？
    答：100 Continue继续，一般在发送post请求时，已发送了http header之后服务端将返回这个信息，表示确认，之后发送具体的信息；
    200 OK 正常地返回信息；201 Created 请求成功，并且服务器创建了新的资源； 202 Accepted 服务器已经接受请求，但尚未处理；301 Moved Permanently 网页已经永久移动到新位置；302 Found 临时性重定向；303 Not Modified 自从上次请求过之后请求的网页未修改过；400 Bad Request 服务器无法解析请求的格式，客户端不应该尝试再次使用相同的内容发起请求；401 Unauthorized 请求未授权；403 Forbidden 禁止访问；404 Not Found 找不到与URI相匹配的资源；500 Internal Server Error 最常见的服务端错误；503 Service Unavaliable 服务端暂时无法处理请求（可能是由于过载或者维护）

浏览器分类，引擎，兼容相关

  1. 常用哪几种浏览器进行测试？有那些内核（Layout engine）
    答：IE：trident内核 Firefox: gecko内核 Opera: blink内核 safari: webkit内核 Chrome: blink内核

  2. 哪些常见的操作会造成内存泄漏？
    答：内存泄漏是指任何对象在你不再拥有或者需要它之后仍然存在。浏览器的垃圾回收器定期扫描对象，并计算引用了每个对象的其他对象数量，如果一个对象的引用数量为0（没有其他对象引用过该对象），或者该对象唯一引用是循环的，那么该对象的内存即可回收；setTimeout的第一个参数使用字符串而非函数，会引发内存泄漏。闭包，循环（在两个对象间彼此引用而且彼此保留时，就会产生一个循环）

  3. 线程和进程的区别？
    答：一个程序至少有一个进程，一个进程至少有一个线程。线程的划分尺度小于进程，使得多线程程序的并发性高。另外，进程在执行过程中拥有独立的内存单元，而多个线程共享内存，从而极大地提升程序的运行效率；
    线程在执行过程中和进程还是有区别的：每个独立的线程有一个程序的入口、顺序执行序列和程序的出口。但是线程不能独立执行，必须依存在应用程序中，由应用程序提供多个线程执行控制。从逻辑角度看，多线程的意义在于一个应用程序中，有多个执行部分可以同时执行。但是操作系统并没有将多个线程看成多个独立的应用，来实现进程的调度和管理以及资源分配。

网站优化性能，SEO

  1. 有哪些网站性能优化的方法？
    答：（1）通过减少http请求的次数：CSS Sprites；js、css源码压缩；图片大小控制合适；网页Gzip；CDN托管；data缓存；图片服务器 （2）前端模板 JS+数据，减少由于HTML标签导致的带宽浪费，前端用变量保存ajax请求结果，每次操作本地变量，不用请求，减少请求的次数 （3）用innerHTML代替DOM操作，减少DOM操作次数，优化javascript性能 （4）当需要设置的样式很多时设置className而不是直接操作style （5）少用全局变量，缓存DOM节点查找的结果，减少IO读取的操作 （6）避免使用css expression（css表达式）或者css动态属性 （7）图片预加载，将样式表放在顶部，将脚本放在底部，加上时间戳

  2. 什么叫优雅降级和渐进增强？
    答：优雅降级：一开始就构建完整的功能，然后再针对低版本的浏览器进行兼容；渐进增强：针对低版本的浏览器进行构建页面，保证最基本的功能，然后再针对高级浏览器进行交互改进和追加功能达到更好的交互体验；区别：优雅降级是从复杂的现状开始，减少用户体验的供给，而渐进增强是从一个非常基础的版本，开始不断的扩充，适应未来的发展需要。

HTML语言

  1. 行内元素和块级元素的区别？行内块元素在IE8下的兼容方法？
    答：行内元素会在一行水平排列，不包含块级元素，设置width，height无效，设置margin上下，padding上下无效；块级元素会各占一行垂直方向排列，和行内元素正好对应。
    行内块元素的兼容，设置display:inline-block; *display:inline; *zoom:1

  2. <!doctype>作用？标准模式和兼容模式
    答：<!doctype>即DTD声明，用于告诉浏览器的解析器用什么文档标准来解析这个文档，这个DTD不存在，浏览器会导致文档以兼容模式呈现。标准模式下，页面渲染和js运行模式都是以该浏览器支持的最高标准运行，兼容模式下，页面以宽松向后兼容的方式显示，模拟老浏览器的行为防止站点无法进行工作。

  3. html5为什么只需要写<!doctype>，不需要写DTD引用？
    答：html5不基于SMGL，所以不需要进行DTD应用，但是要用<!doctype>来规范浏览器行为。

  4. html5有哪些新特性？如何处理html5新标签的浏览器兼容问题？如何区分html和html5？
    答：html5已经不再是SGML的子集，主要是关于图像，位置，储存，多任务等功能的增加。
    （1）绘画canvas （2）用于媒介回放的video和audio元素 （3）用于本地离线存储localStorage长期存储数据，浏览器关闭后数据不丢失 （4） sessionStorage 的数据在浏览器关闭后自动删除 (5) 语义化更好的内容元素 比如 article，footer ，header，nav，section （6）表单控件相关属性 calendar，date，time，email，url，search （7）新的技术 webworker websocket Geolocation ;兼容处理：IE6/7/8支持通过document.createElement的方法来产生新标签，让这些浏览器支持新标签后，还需要添加标签默认样式；也可以直接使用成熟的框架来处理兼容，比如：html5shim一般要配合使用IE的条件注释 （cc:ie6）

  5. 简述对html语义化的理解？
    答：html语义化使页面内容结构化，便于阅读和维护；搜索引擎的爬虫依赖html标记来确定上下文和各个关键字的权重，利于seo

CSS语言

  1. 清除浮动的哪些方式？比较好的是哪一种？
    答：（1）父级元素定义height （2）结尾处添加空div，clear:both （3）父级div定义伪类 after，before和zoom （4）父级div定义overflow：hidden （5） 父级div定义overflow: auto （6）父级div也浮动，要定义宽度 （7）结尾处加br标签 clear:both （8）父级div定义伪类after，before，zoom，设置display:table
    比较好的是第3种和最后一种

  2. box-sizing常用的属性有那些？ 分别有什么作用？
    答： box-sizing: content-box || border-box || inherit ; content-box 高度和宽度分别应用到元素的内容框。 border-box 元素指定的任何内边距和边框都在设定的高度和宽度内绘制。inherit 是默认样式

  3. link和@import导入样式的区别？
    答： link是XHTML4的标签，除了加载CSS，还能用于定义RSS，连接属性rel等，@import只能用于导入css； @import引用的CSS会有阻塞问题，只有页面全部加载完再加载。

JS语言

  1. 介绍js的基本数据类型？
    答：Undefinded Null String Number Object Boolean

  2. js 有哪些内置的对象？
    答：数据封装类对象：Object Array Boolean Number String
    其他对象： Function，Arguments，Math，Date，RegExp，Error

  3. this对象的理解？
    答：this指向函数的直接调用者，而非间接调用者；如果有 new 关键字，this指向new出来的对象； 在事件中，this指向触发这个事件的对象，IE中的attachEvent中的this总是指向全局对象Window

  4. eval是做什么的？
    答：eval是把对应的字符串解析成js代码并运行；应该尽量避免使用eval，因为它并不安全（脚本嵌入式攻击），而且非常消耗性能（2次，一次解析成js语句，一次执行js）； eval还用在把JSON字符串转换成JSON对象的时候，比如: var obj = eval(‘(‘+str+’)’)

  5. ★★★DOM怎样添加，移除，移动，复制，创建和查找节点？
    答：创建节点：createDocumentFragment()创建一个DOM片断 createElement() 创建一个具体的元素 createTextNode() 创建一个文本节点； 添加，移除，替换，插入分别对应是appendChild() removeChild() replaceChild() insertBefore() 在已有的子节点之前插入一个新的子节点； 查找的方法有：getElementsByTagName()通过标签名进行查找 getElementsByClassName() getElementsByName() getElementById()

  6. null 和 undefined的区别？
    答： null是一个空对象，转为数值是0；undefined是一个表示‘无’的原始值，转为数值是NaN
    （1）变量被声明了，但没有赋值就是undefined (2)调用函数时，应该提供的参数没有提供，改参数是undefined （3）对象没有赋值的属性是undefined （4）函数默认没有返回值时是undefined
    （1）作为函数的参数，表示该函数的参数不是对象，比如在node异步回调函数中对数据的处理（2）作为对象原型链的终点

  7. new操作符具体做了什么？
    答：（1）开辟内存空间创建空对象，并且把this值指向该对象，同时继承了该构造函数的原型 （2） 属性和方法被添加到this引用的对象 （3）最后隐式地返回this

  8. JSON的了解？
    答：JSON（Javascript Object Notation）是一种轻量级的数据交换格式，它是基于js的一个子集。数据格式简单，易于读写，占用带宽小。格式：采用键值对的方式 比如{“age”:12,”name”:”back”}

  9. call() 和 apply() 的区别和作用？
    答： apply()函数有两个参数，第一个参数是上下文，第二个参数是参数组成的数组。如果上下文是null，则使用全局对象代替。
    如 function.apply(this,[1,2,3])
    call()函数的第一个参数是上下文，后续是实例传入的参数序列 如 function.call(this,1,2,3)

  10. 如何获取UA？
    答：navigator.appName/appVersion/appCodeName/userAgent
