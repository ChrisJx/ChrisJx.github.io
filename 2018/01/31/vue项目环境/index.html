使用vue-cli来构建自己的项目

  1. 第一还是npm 的版本问题，npm -v 查看版本 （5.4.0）
    如果要查看最新npm的版本 npm view npm version
    如果要升级到最新的版本 npm install -g npm
    如果要升级到特定的版本 npm install -g npm@5.4.0

  1. 全局安装vue-cli
    npm install vue-cli -g
    如果想要查看已经安装的vue-cli的版本号，vue -V（2.8.2） 这也是vue的版本号
    如果想要更新到最新版本的vue-cli
    npm update vue-cli
    查看一下当前全局 vue-cli 的版本
    npm view vue-cli

  2. 基于vue初始化一个webpack项目
    vue init webpack-simple projectName
    参数 表示模板的名称 vue-cli官方为我们提供了5种模板：
    webpack: 一个全面的webpack+vue-loader的模板，功能包括热加载，linting,检测和css拓展
    webpack-simple: 一个简单的webpack+vue-loader 模板，不包含其他功能，用于快速搭建
    ……
    参数 <project-name>：标识项目名称，这个你可以根据自己的项目来起名字。

输入命令后，会询问我们几个简单的选项，我们根据自己的需要进行填写就可以了。
Project name :项目名称 ，如果不需要更改直接回车就可以了。注意：这里不能使用大写，所以我把名称改成了vueclitest
Project description:项目描述，默认为A Vue.js project,直接回车，不用编写。
Author：作者，如果你有配置git的作者，他会读取。
Install vue-router? 是否安装vue的路由插件，我们这里需要安装，所以选择Y
Use ESLint to lint your code? 是否用ESLint来限制你的代码错误和风格。我们这里不需要输入n，如果你是大型团队开发，最好是进行配置。
setup unit tests with Karma + Mocha? 是否需要安装单元测试工具Karma+Mocha，我们这里不需要，所以输入n。
Setup e2e tests with Nightwatch? 是否安装e2e来进行用户行为模拟测试，我们这里不需要，所以输入n。

  1. 把文件切换到初始化的项目中
    cd projectName

  2. 安装依赖
    npm install
    或者使用淘宝镜像cnpm 来安装，这样会快很多

  3. 运行项目
    npm run dev

  4. webstorm打开pc项目，由于依赖项非常的多编辑器在检索module文件时会非常的耗内存，
    所以在用webstorm编辑器启动项目的时候我们往往会在node-module文件夹右键去阻止检索该文件，
    本来在make-directory中的选项中有excluded选项，但是却找不到了…

  5. 项目的目录参考
    项目目录结构如下：
    build config 这两个目录是webpack以及node服务器的配置。
    node_modules 这个目录是npm install命令执行后按照的依赖项所在目录。
    src 是用来存放项目代码的，我们的编码也都是在这个目录里进行的。
    .babelrc babel配置文件，把我们ES2105的代码通过它编译成ES5的。
    .editorconfig 编辑器配置。
    .eslintignore 忽略语法检查的目录文件配置。
    .eslintrc .jseslint 的配置文件。
    .gitignore 配置Git仓库的忽略。
    index.html 项目入口模板文件。
    package.json node配置文件

  6. 有些配置文件不一样，尝试更新了下webpack版本。npm install webpack -g
    npm install webpack@1.31.x -g
    webpack的信息 npm info webpack
    查看自己webpack 的版本 webpack --help 找到 webpack -v (3.9.1)

  7. 项目中需要使用到sass, 所以安装 sass-loader
    cnpm install sass-loader --save-dev
    package.json文件下会生成sass-loader的版本信息
    由于sass-loader是依赖node-sass所以我们还要安装node-sass

然后在webpack.base.conf.js 的ruler中添加

  1. 项目中需要用到ajax请求，安装 vue-resource
    npm install vue-resource --save
