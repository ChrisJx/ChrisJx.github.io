app调试

发表于 2018-01-30 | 分类于 webview

和app调试的过程中，遇到两次”webview”作怪的问题，很明显在 QQ 的webview中可以很顺利的打开，
但是在我们app中却很怪戾。

  • 首先来一段 js 调用app中查看图片的code

用类似的方法调试过一个调用原生的发帖，但是没有做好 android 和 ios 的方法统一和命名规范，以后要注意这点。

  • webview中的 script 中的内容会报错，并且会影响 webview 中样式的加载问题。

测试的方法非常原始，从最简单的 html 开始，一段一段内容的往上加，加到 script 标签的时候，发现页面渲染
出现了问题，后来发现webview 无法获取 document.documentElement.width 这个属性，并且页面加载前无法加载
这段js。和上次ios中webview无法获取 网页的内容宽度类似。

正确的调试方法：

  1. 手机连接电脑
  2. 打开chrome 浏览器，地址栏上输入 chrome://inspect
  3. 手机上打开app中的 webview，并且app要打开调试开关

思考：
1、webview 中的调试一般通过打印或者抓包的方法来调试，效率比较低，不能通过直接调试样式或者打断点的方式进行
调试。所以inspect在浏览器中调试可以提高不少效率，但是踩得坑也不少。
2、所有的细节都需要 啃透 ，没有花费时间 *啃透 的东西，都不属于自己的沉淀，只会沦为时间的过客。

  • 如果在 inspect 没有办法实现的情况下，可以通过抓包代理的方式来调试

    实现的方式有些曲折，php_study 通过配置代理访问本地的项目(本地 host 配置) –> 请求远端的服务器（已经做了跨域的处理） –>
    Fiddler 配置代理把电脑上的所有的请求都拦截，准确来说应该是 以这台电脑的ip 发送的所有请求，包括本地服务
