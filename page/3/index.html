  • 原型对象中的 construct属性指向创建原型的这个构造函数；
  • Function函数本身也是它自己的实例对象（也就是Function.proto===Function.prototype）

SSH

SSH是一种网络协议，用于计算机之间的加密登录，我们后面用到的git客户端也集成了SSH

  • 基本格式： ssh user@host

​ user 代表真实存在的用户，host代表要登录的远程计算机

  • 常见的加密技术： 对称性加密和非对称性加密，SSH属于后者

​ 对称加密算法在加密和解密时使用的是同一个密钥；而非对称加密算法需要两个密钥来进行加密和解密，这两个秘钥分别是公开密钥（public key，简称公钥）和私有密钥（private key，简称私钥）。

  • 工作原理

​ 公钥和私钥是成对出现，可以通过ssh-keygen -t rsa来创建，既可以通过密钥来加密数据，也可以通过私钥来加密数据，如果是以公钥进行的数据加密，只能与之相对应的私钥才可以解密，相反如果以私钥进行的数据加密，则只能与之对应的公钥才可以将数据进行解密，这样就可以提高信息传递的安全性。

  • 免密码登录的应用

​ 我们可以将本地机器上的公钥保存到特定的远程计算机上，这样当我们再次登录访问这台远程计算机时就可以实现免密码登录了。

  • SSH通信过程

    1、客户端发送请求
    2、服务器接收到请求之后，把服务器的公钥发送给客户端
    3、客户端收到公钥后会存储在某个位置（C:\Users\用户名.ssh）
    4、客户端再次发送登录请求（密码会使用公钥加密）
    5、服务器接收到密文后使用私钥进行解密然后验证是否正确

实际操作

1、ssh-keygen -t rsa会创建公钥和密钥（默认在用户目录/.ssh目录下）
2、ssh-copy-id user@host添加到对应远程主机的用户目录/.ssh目录下
3、也可以登录远程主机，进入到用户目录/.ssh目录下手动创建authorized_keys文件，并将自已的公钥粘入该文件。

关于版本控制

版本控制（Version Control Systems）是一种记录一个或若干文件内容变化，以便将来查阅特定版本修订情况的系统。这个系统可以自动帮我们备份文件的每一次更改，并且可以非常方便的恢复到任意的备份（版本）状态。

文档参阅

实现版本控制的软件有很多种类，大致可以分为本地版本控制系统、集中式版本控制系统、分布式版本控制系统。

  • ​集中式版本控制系统

实际开发环境，一个项目通常是由多人协作共同完成的，如何让在不同终端上的开发者协同工作成了亟待解决的问题，集中式版本控制系统便应运而生了。它通过单一的集中管理的服务器，保存所有文件的修订版本，协同工作的开发者都通过客户端连到这台服务器，取出最新的文件或者提交更新。其代表为SVN，如下图所示。

这种方式很好解决了多人协同开发的问题，但是也有一个弊端，如果集中管理的服务器出现故障，将会导致数据（版本）丢失的风险，另外协同开发者从集中服务器中更新数据时，严重依赖网络，如果网络不佳，也给开发带来诸多不便。

  • 分布式版本控制系统
    分布式版本控制系统，则不需要中央服务器，每个协同开发者都拥有一个完整的版本库，这么一来，任何协同开发者用的服务器发生故障，事后都可以用其它协同开发者本地仓库恢复。
    由于版本库在本地计算机，也便不再受网络影响了。如果要将本地的修改，推送给其它协同开发者，还需要一台共享服务器，所有开发者通过这台共享服务器同步和更新数据。如下图所示。

分布式版本控制系统弥补了前面两种版本控制系统的缺陷，成为了版本控制的首选方案。其代表就是Git.

参考资料 Git教程

git是分布式版本控制系统

  • git中的三个重要区域
  1. 工作目录
    是对项目的某个版本独立提取出来的内容。这些从Git仓库的压缩数据库中提取出来的文件，放在磁盘上供你使用或修改。
  2. 暂存区
    是一个文件，保存了下次将提交的文件列表信息，一般在Git仓库目录中。有时候也被称作“索引”（Index），不过一般说法还是叫暂存区域。
  3. git仓库
    Git用来保存项目的元数据和对象数据库的地方。 这是Git 中最重要的部分，从其它计算机克隆仓库时，拷贝的就是这里的数据。
  4. git本地仓库
    Git本地仓库指的是开发者计算机中的仓库。

  • 创建版本库

版本库又名仓库，英文名是respository,可以简单理解成一个目录，这个目录里面的所有文件都可以被git管理起来，每个文件的修改，删除，git都能跟踪，以便在任何时刻追踪历史，或者在将来某个时刻还原。

我们如果想要利用git进行版本控制，需要将现有项目初始化为一个仓库，或者将一个已有的使用git进行版本控制的仓库克隆到本地。

git init会在当前的项目目录中创建一个名为.git的隐藏目录，这个目录包含了暂存区和仓库两个区域，有了这个隐藏目录就可以使用git来管理项目了，可以通过ls -al来查看文件下的内容

建立仓库后就可以进行开发了，进入到仓库目录，添加我们需要的文件，通过git status可以检测当前仓库文件的状态（未追踪 untracked）

git add 文件名/文件路径 git add * 或者git add -A 代表添加所有的文件到暂存区

Git远程（共享）仓库

通过上面学习我们可以很好的管理本地版本控制了，可是如果我们下班回到家里突然来了灵感觉得有部分代码可以优化，如果能接着公司电脑上的代码继续写就好了。
另一种情形，假设项目比较大，不同的功能模块由不同的开发人员完成，不同模块儿之间又难免会依赖关系，这时如果我们的代码互相合并（融合）就好了。所有模块开发完毕后，需要整合到一起，要能做到准确无误是多好。

解决方法：
借助一个远程仓库，大家可以共享代码、历史版本等数据，便可以解决以上遇到的所有问题。

  • 创建共享仓库

初识原型

  • 对象的本质：无序的键值对的集合
  • 原型：函数都会有一个原型属性prototype,它本质上是Object的一个实例
  • 原型的作用：实现数据共享（多个实例对象共享数据）；实现继承；开发中减少命名冲突，内存占用减少

  • 相同的对象（即使内容相同），但是在内存中是独立的空间，地址不相同

  • __proto__构造函数的实例对象都会有一个属性__proto__,它不是标准的属性，不能在编程中使用，该属性只能在浏览器内部使用，该属性和构造函数中的prototype指向相同的对象

总结：prototype在构造函数中，__proto__在实例对象上，它们指向同一个对象

jQuery中获取元素内容宽度的方法

  • jDom.innerwidth() 获取元素内容加上内边距的宽度和
  • jDom.outerWidth() 获取元素内容 加上内边距 加上边框的宽度和
  • jDom.outerWidth(true) 获取元素内容 加上 内边距 加上边框 加上外边距的宽度和
    注意一些细节:
  1. jDom.outerWidth()不加参数的形式获取元素宽度的方法和js中的dom.offsetWidth相同。
  2. 当盒模型的不同，即box-sizing分别设置为border-box和content-box时，宽度的计算方式相同。

php语言基础（一）

2015-05-10

基础语法

  • hello world

  • 文件名
    文件以.php后缀结尾，所有程序包含在
    <?php /** 这里是代码 **/ ?>

  • 变量
    1、变量以$开头 字母/数字/下划线 不能以数字开头
    2、大小写敏感（区分大小写）

  • 数据类型
    字符型 整形 浮点型 布尔型 数组（索引数组和关联数组）对象 resource NULL

  • 单引号和双引号的区别
    php中，双引号中的变量($var)和特殊字符(\r\n之类)会被转义，单引号中的内容不会被转义（所以效率更高）。

  • 内容输出
    echo 输出简单数据类型，如字符串、数值
    print_r() 函数 输出复杂数据类型，如数组、对象
    var_dump() 函数 输出对象的详细信息，如数组、对象

  • 运算符
    基本和js中的运算符保持一致
    注意:
    在php中表示字符串拼接用.号，js中使用+号

  • 函数
    和js基本一致，注意php中的函数名对大小写不敏感，可以有默认的参数

  • 分支 循环语句
    和js中基本一致

  • 表单处理
    表单中的name属性，是用来提供给服务端接受所传递的数据所接受的
    表单中的action属性，是用来设置接受数据的处理程序
    表单中的method属性，是用来设置发送数据的方式
    注意：
    当上传文件时，需要设置enctype="multipart/form-data",并且只能以post方式发送数据
    $_GET[] 用来接受get方式发送的数据
    $_POST[] 用来接受post方式发送的数据
    $_FILES[] 用来接受文件的上传

  • 常见的PHP函数
    in_array() 检测元素是否在数组中（第一个参数是待检测的元素，第二个元素是数组）
    count() 计算数组的长度
    array_key_exists() 检测数组中是否存在key（第一个参数是待检测的key，第二个元素是数组）
    file_get_contents() 把整个文件内容读取成一个字符串（接受的参数是文件的路径）
