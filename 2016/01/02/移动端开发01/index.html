移动端开发（一）

2016-01-02

移动端开发的优势

主要有以下三个方面的原因使得现在的移动web开发愈来愈热

  1. 智能手机的普及，手机端浏览网页较为快捷方便
  2. 电商在手机平台上的销售额占据主要，移动支付已经成为主流
  3. 最新版本的浏览器，有效地利用H5和C3，避免很多的兼容性问题

移动端开发的基础知识

  • 流式布局
    就是百分比布局，根据盒子的宽度设置成百分比，从而根据屏幕的宽度来进行伸缩，
    不受固定像素的限制，内容向两侧填充.

  • viewport（视口）
    pc页面在移动设备上显示的情况，默认是缩放显示的；
    移动端用来承载网页的这个区域，就是我们的视觉窗口，viewport（视口）,这个区域可以设置
    高度和宽度，可以按比例放大和缩小，而且能设置是否允许用户自行缩放。

  • width 宽度设置的是viewport宽度，可以设置为 device-width特殊值
  • height 一般不需要设置
  • initial-scale 初始缩放比，大于0的数字
  • maximum-scale 最大缩放比，大于0的数字
  • minimum-scale 最小缩放比，大于0的数字
  • user-scalable 用户是否可以进行缩放，yes或no（1或0）

  • Retina屏
    retina是一种显示技术，可以把更多的像素点压缩在一块屏幕里，从而达到更高的分辨率，
    并提高屏幕显示的细腻程度。
    物理像素 -> 像素点
    屏幕尺寸（像素） -> 1px

  • 移动web网页首页的切图
    如果是在1:1地显示在移动设备当中，图标会失真；在行业中通用的设计稿尺寸 640px；
    首页的布局是以百分比布局为主，定义最大宽度和最小宽度的布局，这种布局适用的网页：
    首页，门户，电商等

  • 移动端事件
    Touch事件组：
    touchstart: 当手指触碰屏幕时发生，而不管是多少只手指
    touchmove: 当手指在屏幕上连续滑动时触发；通常我们在滑屏页面的时候，可以调用
    event.preventDefault()来阻止默认情况的发生，防止页面滚动。
    touchend: 当手指离开屏幕时触发
    touchcancel: 系统停止跟踪触摸时触发，例如在触摸过程中突然页面alert()一个提示框，
    此时会触发该事件，这个事件比较少用

  • 触摸事件的响应顺序：

  1. ontouchstart
  2. ontouchmove
  3. ontouchend
  4. onclick 300ms延时

  • Event
    originalEvent: 原生事件，是jq封装的一个事件
    targetTouches: 屏幕目标元素上所有的触摸点数
    changedTouches: 屏幕上触摸点改变的个 数（包括从无到有从有到无和位置改变）
    touches: 屏幕上所有的的触摸点

  • 获取第一个触摸点相对可视区域的坐标touches[0].clientX

  • 动画结束事件
    动画结束事件是transitionend
    应用场景
    动画和过渡结束的时候都会触发animationend 和 transitonend 事件，h5制作轮播图的过程中，使用动画结束事件，瞬间改变index的值，达到无缝轮播的过程，解决了之前js轮播图的时间限制。

注意：
在touchend事件中，event只会记录到changedTouches的个数
一般情况下用addEventListener()绑定事件

  • css样式中的属性

  • 点击屏幕清除高亮效果

  • 清除表单控件的立体显示效果

  • 防止内容溢出，改变盒模型

  • 为了解决移动端图标失真问题，把精灵图尺寸压缩一倍，相应地在设置图标的位置时也需要除以2

  • map文件
    通常使用css和js 压缩的过程。如果出错 显示浏览会提示那一行出错Map就是提供了记录代码位置的文件

  • bootstrap各个版本的优劣
    2.x.x 停止维护
    优点：兼容性好
    缺点：代码不够简洁，功能不够完善。
    3.x.x 目前使用最多
    稳定，但是放弃了IE6-IE7。对IE8支持但是界面效果不好。
    偏向用于开发响应式布局、移动设备优先的 WEB 项目。
    4.x.x 测试阶段
    更偏响应式，移动设备，代码更精简。

各种布局汇总

布局问题基本上在移动端和pc端都是首先要考虑的，也就是达成适配各种屏幕的效果

  • 响应式布局
    所谓响应式布局就是流式布局+媒体查询，流式布局用来解决不同宽度的布局问题，外加媒体查询，可以调整布局，例如大屏幕是布局1，小屏幕是布局2，这种布局通吃pc和移动端，做到精细处，两者的效果都很好，
    缺点是媒体查询是有限的，也就是可以枚举出来的，只能适应主流的宽高。

  • rem布局
    参考：http://isux.tencent.com/web-app-rem.html
    原理是，先按定高宽设计出来页面，然后转换为rem单位，配合js查询屏幕大小来改变html的font-size，最终做出所谓的完美自适应。
    pc端的方法：

  1. 借助@media screen来进行媒体屏幕查询，针对不同屏定义不同的响应式容器
  2. 还有就是设置固定宽度也就是min-width,小于这个宽度就出现滚动条，大于这个宽度内容居中外加背景
    移动端的方法：
  3. 移动端是通过百分比和最大宽度最小宽度，并利用meta标签来达到适配不同的手机终端
  4. 响应式布局
  5. js加上rem布局（比如淘宝）

响应式开发的优势和劣势

  • 优点很明显：面向几乎所有的终端，比移动端开发支持度更好，有效减少后台的数据交互，新的网站一般都会优先考虑响应式开发
  • 劣势是：开发效率低，成本大

开发响应式网站

项目结构

  • taobao………..项目文件夹
    • js……………….js文件目录
    • lib………………第三方框架目录
    • fonts…………….字体文件目录
    • images……………图片文件目录
    • css………………css文件目录
    • favicon.ico……….网站图标
    • index.html………..站点入口

HTML约定

  • 在head中引入必要的CSS文件，优先引用第三方的CSS，方便我们自己的样式覆盖。
  • 在body末尾引入必要的JS文件，
    优先引用第三方的JS，
    注意JS文件之间的依赖关系，
    比如bootstrap.js依赖jQuery，
    那就应该先引用jquery.js 然后引用bootstrap.js。

CSS约定

  • 除了公共级别样式，其余样式全部由 模块前缀
  • 尽量使用 直接子代选择器，少用间接子代 避免覆盖

使用bootstrap常用样式

container类

  • 用于定义一个固定宽度且居中的版心

栅格系统

  • Bootstrap中定义了一套响应式的网格系统，
  • 其使用方式就是将一个容器划分成12列，
  • 然后通过col-xx-xx的类名控制每一列的占比

row类

  • 因为每一个列默认有一个15px的左右外边距
  • row类的一个作用就是通过左右-15px屏蔽掉这个边距

col-\-*类

  • col-xs-[列数]：在超小屏幕下展示几份
  • col-sm-[列数]：在小屏幕下展示几份
  • col-md-[列数]：在中等屏幕下展示几份
  • col-lg-[列数]：在大屏幕下展示几份
  • xs : 超小屏幕 手机 (<768px)
  • sm : 小屏幕 平板 (≥768px)
  • md : 中等屏幕 桌面显示器 (≥992px)
  • lg : 大屏幕 大桌面显示器 (≥1200px)

可以结合push pull offset使用

hidden 类

hidden-xs,hidden-sm,hidden-md,hidden-lg在不同的屏幕下隐藏。

text-* 类

text-center 文本居中
text-left 文本左对齐
text-right 文本右对齐

pull-* 类

pull-left 左浮动类
pull-right 右浮动类

center-block 类

让一个固定宽度的元素居中。
