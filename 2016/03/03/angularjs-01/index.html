AngularJS 框架学习（一）

2016-03-03

Angular介绍

  • AngularJS是一款由Google公司开发维护的前端MVC框架，其克服了HTML在构建应用上的诸多不足，
    从而降低了开发成本提升了开发效率。

  • MVC (Model View Control) 是一种开发模式.

    • 模型（Model）用来处理数据（读取和设置），一般指操作数据库
    • 视图（View）用来展示数据，一般是指HTML页面的展示
    • 控制器（Controller）用来传输数据，一般用作连接模型和视图的桥梁

    如图：

  • AngularJS和之前的jQuery属性的区别？

    `jQuery`准确来说是一个**类库**（一系列函数的集合），以`DOM`作为驱动的核心，而`AngularJS`则是一个**框架**（很多类库的集合），以*数据*和*逻辑*作为驱动的核心。

  • AngularJS的特性

    最为核心的特性是： 模块化，双向数据绑定，语义化标签，依赖注入

  • 与AngularJS类似的框架

    `Backbone` `KnockoutJS` `Vue` `React`

Angular Js 的基本结构

  • AngularJS构建应用（App）时是以模块化（Module）的方式组织的，即将整个应用划分成若干模块，
    每个模块都有各自的职责，最终组合成一个整体。

  • 定义应用
    在任意的DOM元素上使用一个属性ng-app就可以定义一个Angular JS的应用，
    ng-app属性所在的DOM元素所包含的所有子元素都属于此应用的一部分。

  • 定义模块
    引入Angular框架后，会从中提取一个全局的对象Angular,在此对象下有很多方法，angular.module()
    可以实例一个模块。

注意： 应用APP本质上也是一个模块，一个比较大的模块而已。

  • 在上述步骤后,,创建控制器(controller)

  • 在HTML标签中导入module和controller名称，从而建立连接模型（Model）和视图（View）的桥梁，
    通过双花括号来调用数据。

指令

HTML语言本身在构建应用的时候存在缺陷，没有变量，没有分支判断，没有循环等，为了弥补这些缺陷，
AngularJS拓展了一些HTML的属性和标签，也就是指令（不同的指令，让标签执行不同的业务），比如
ng-repeat ng-controller

  • 关于无值属性
    jquery 中attr()和prop()属性的区别？
    通过attr()方法无法获取html中布尔值属性本身的布尔值，获取到的是checked字符串或者undefined；
    prop()方法可以获取相关布尔值属性的布尔值

  • 内置指令

    include存在跨域方面的问题，另外不能绑定双花括号数据

  • 自定义指令
    AngularJS 提供很多指令，但是并不能满足所有开发者的所有需要,AngularJS 允许开发者根据自已的实际业务扩展指令
    directive()接受两个参数：1.指令名称 2. 回调函数
    返回的可以是一个函数或者对象（多数情况下）

数据绑定

  • AngularJS是以数据作为驱动的MVC框架，所有的模型（Model）里的数据都是通过控制器（Controller）
    展示到视图（View）中。数据绑定就是把模型中的数据和视图中的数据进行关联，分为单向绑定和双向绑定两种。

  • 将获取的数据通过一系列的方法展示到特定的位置给用户，数据跟视图的位置存在一一对应的关系，
    这种一一对应的关系就叫做数据绑定。实际上就是atrTemplate完成的功能，这是单向绑定的情况。

注意：
ng-bind和双花括号这两个指令都可以实现模型的数据向视图模板的绑定。区别是通过双花括号方式绑定数据会出现闪烁的现象，
原因是html文档的解析顺序（可能angular还没有加载，html中的数据已经被读取解析）。还可以通过ng-cloak为元素
添加隐藏指令，这样直到angular解析到这个元素时，这个元素才会显示。

  • 通过为表单元素添加ng-model指令来实现视图（View）向模板（Model）数据的绑定。

脏值检查

$scope.$digest()触发脏值检查
AngularJS进行双向数据绑定的时候，当模型中的数据发生改变时，就会成为“脏值”。如果不手动地进行脏值检查，不会自动更新视图（这样会浪费性能）。
注意：
事件指令内部会自动触发一次脏值检查

AngularJS的作用域

AngularJS每嵌套一个控制器就相当于嵌套了一个函数，类似于函数的作用域链。
每个控制器（Controller）又都对应一个模型（Model）也就是$scope对象，不同层
级控制器（Controller）下的$scope便产生了作用域。

  • 根作用域
    一个AngularJS的应用（ng-app）会自动创建一个根作用域$rootScope,这个根
    作用域在整个应用内都可以被访问到。
  • 子作用域
    通过ng-controller可以创建一个字作用域，子作用域可以访问父作用域的数据。

常见的错误：

  • 如果在html标签中创建了ng-controller的指令，那么必须在应用中事先定义这个控制器。

过滤器

AngularJS中使用过滤器格式化地展示数据，在双花括号中用|来调用数据，用:来传递参数。

  • 内置过滤器
  • currency 将数据格式化货币格式 (|currency:'$':3)
  • date 将数据格式化日期格式 (|date:'yyyy-MM-dd hh:mm:ss')
  • lowercase 将文本转换成小写 (|lowercase)
  • uppercase 将文本转换成大写 (|uppercase)
  • number 数字格式化，能控制小数的位数 (|number:3)
  • limitTo 取出字符串或者数组的前几位或者后几位 (|limitTo:-2)
  • orderBy 对数组或对象进行排序,默认按字母或者数字排序（第一个参数是排序标准,第二个参数是布尔值，控制方向,true是倒序）
    (|orderBy:'':true) 或 (|orderBy:'age':false)
  • json 将javascript对象转换成json字符串 (|json)
  • filter 在数组中选择满足条件的子集，返回新数组，条件可以是函数 (|filter:'s')

输出的结果是：

  • 自定义过滤器
    注意两个return

  • 把字符串转换成以指定个数的大写开头的字符

注意：
过滤器可以链式操作，比如|lowercase|limitTo:2

  • $interval服务可以实现定时器功能，$interval.cancle(stop)清空定时器，timeout也一样

  • $http服务会发起一个异步请求，服务端返回数据，将数据跟新到视图上

客户端以特定的格式传递数据，服务端也要以特定的格式来接受数据。在PHP中$_POST[]是专门用来接受

配置块

运行块

可以直接运行AngularJS中的相关服务
App.run([‘$rootScope’,function($rootScope){
$rootScope.name = ‘chris’;
}])

不但如此，run方法还是最先执行的，利用这个特点我们可以将一些需要优先执行的功能通过run方法来运行，比如验证用户是否登录，未登录则不允许进行任何其它操作。

路由

一个应用是由若干个视图组合而成的，根据不同的业务逻辑展示给用户不同的视图，路由就是实现这一功能的。

  • SPA（Single Page Application）指的是通单一页面展示所有功能，通过Ajax动态获取数据然后进行实时渲染，结合CSS3动画模仿原生App交互，然后再进行打包（使用工具把-Web应用包一个壳，这个壳本质上是浏览器）变成一个“原生”应用。
  • 在PC端也有广泛的应用，通常情况下使用Ajax异步请求数据，然后实现内容局部刷新，局部刷新的本质是动态生成DOM，新生成的DOM元素并没有真实存在于文档中，所以当再次刷新页面时新添加的DOM元素会“丢失”，通过单页面应可以很好的解决这个问题。

  • AngularJS中的路由
    在后端开发中通过URL地址可以实现页面（视图）的切换，但是AngularJS是一个纯前端MVC框架，在开发单页面应用时，所有功能都在同一页面完成，所以无需切换URL地址（即不允许产生跳转），但Web应用中又经常通过链接（a标签）来更新页面（视图），当点击链接时还要阻止其向服务器发起请求，通过锚点（页内跳转）可以实现这一点。AngularJS中把
    锚点的变化封装成路由，这是和后端路由的根本区别。

  • 使用路由

  1. 引入angular-route.js

  2. 实例化模块（App）

  3. 配置路由模块

  4. 布局模板
    通过ng-view指令布局模板，路由匹配的视图会被加载 渲染到这些区域

  • 路由参数
  1. 提供两个方法匹配路由，分别是when和otherwise，when方法需要两个参数，otherwise方法做为when方法的补充只需要一个参数，其中when方法可以被多次调用。
  2. 第1个参数是一个字符串，代表当前URL中的hash值。

  3. 第2个参数是一个对象，配置当前路由的参数，如视图、控制器等。

    c、controller 视图模板所属的控制器
    d、redirectTo跳转到其它路由

  4. 获取参数
    在控制中注入$routeParams可以获取传递的参数

bower

A package manager for the web
基于NodeJS的一个静态资源管理工具，由twitter公司开发维，解决大型网站中静态资源的依赖问题。

  3. bower search 查找资源信息
  4. bower install 安装（下载）资源，通过#号可以指定版本号
  5. bower info 查看资源信息
  6. bower uninstall 卸载（删除）资源
